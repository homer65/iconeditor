Der IconEditor ist eine in der Programmiersprache Java geschriebene Software. Mit ihr soll es möglich sein Icon's und Bilder zu bearbeiten. Derzeit werden die Typen png,gif,jpg,jpeg und ico unterstützt. Für den ico Typ benutzen wir eine Bibliothek von http://image4j.sourceforge.net.

Der IconEditor kann per Java Webstart von der Seite myoggradio.org aus dem Download Bereich aufgerufen werden.

Die Software kann gemäß den Bestimmungen der GPL in der Version 2 oder 3 genutzt werden. Siehe Unterordner license.

Der Quellcode liegt in Form eines Eclipse Projekt bei.

Die Funktionen um ein Icon zu bearbeiten nennen wir Operation. Der IconEditor hatt eine Undo Funktionalität, mit der es möglich ist eine Operation rückgängig zu machen. Eine Größenänderung - resize - des Icons kann nicht rückgängig gemacht werden. Im Gegensatz zu einer Operation kann ein Util nicht rückgängig gemacht werden. Um eine Operation auszuführen braucht diese als Input eine oder mehrere Farben und ein oder mehrere Pixel. Eine Farbe wählt man durch Klicken auf den Farbauswahlpanel (links oberhalb des Icons). Ein Pixel wird einfach durch clicken auf das Icon ausgewählt.

Derzeit gibt es folgende Operation:
(1) SetPixel - Benötigt eine Farbe und ein Pixel
Ändert die Farbe eines Pixel
(2) FillBox - Benötigt eine Farbe und zwei Pixel
Füllt einen rechteckigen Bereich mit einer Farbe
(3) EraseBox - Benötigt zwei Pixel
Löscht einen rechteckigen Bereich
(4) FillCircle - Benötigt eine Farbe und zwei Pixel
Füllt einen Kreis mit einer Farbe
Allerdings sehen Kreise erst bei hoher Auflösung gut aus
(5) FillEllipse - Benötigt eine Farbe und drei Pixel
Füllt eine Ellispe mit Farbe
Die ersten beiden Pixel sind die Brennpunkte der Ellipse
Das dritte Pixel ist ein Punkt auf dem Rand der Ellipse
Damit kann man auch eine Linie zeichnen, indem man den Rand auf die Verbindungslinie der Brennpunkte setzt
(6) ChangeColor - Benötigt eine Farbe und ein Pixel
Ändert die Farbe aller Pixel, die die gleiche Farbe wie das ausgewählte Pixel haben
(7) ClipBox - Benötigt zwei Pixel
Schneidet aus dem Icon einen rechteckigen Bereich aus, der zum neuen Icon wird
(8) Rotate
Dreht das Icon um 90 Grad gegen den Uhrzeigersinn
(9) ShiftIcon - Benötigt ein Pixel
Die Pixel des Icon sind durchnummeriert. Oben links ist das Pixel 0,0.
Wählt man das Pixel n,m aus, so wird das Icon um n Pixel nach links und m Pixel nach oben verschoben.
Dabei werden Pixel die nach oben bzw links herausfallen unten bzw rechts wieder angehängt.
(10) DrawLine - Benötigt eine Farbe und zwei Pixel
Zeichnet eine Linie vom ersten zum zweiten Pixel
(11) SwitchColor - Benötigt zwei Pixel
Alle Pixel, die die Farbe des ersten Pixel haben bekommen die Farbe des zweiten Pixel.
Alle Pixel, die die Farbe des zweiten Pixel haben bekommen die Farbe des ersten Pixel.
(12) FillPolygon - Benötigt drei oder mehr Pixel und eine Farbe
Aus den n (n>2) Pixeln wird ein geschlossenes Polygon gezeichnet, dessen Inneres
mit einer Farbe gefüllt wird. Die Farbe wird erst akzeptiert, wenn mindestens
drei Pixel vorhanden sind. Die Farbe schließt die Operation ab.
(13) FillBoxWithImage - Benötigt zwei Pixel und ein Image
Das Image wird ins Icon an die durch zwei Pixel markierte Stelle eingefügt.
Dabei wird das Image auf die passende GRöße skaliert.
(14) Heller - Wirkt sofort
Macht das Bild etwas heller
(15) Dunkler - Wirkt sofort
Macht das Bild etwas dunkler
(16) Insert Image - Benötigt ein Pixel
Fügt das Image an die markierte Stelle in seiner Originalgröße ein.
(17) Insert Text - Benötigt ein Pixel und eine Farbe
Es wird ein Dialog aufgerufen, in dem man einen Text und eine Textgröße angeben kann.
Dannach wird der Text an die markierte Stelle eingefügt

Derzeit gibt es folgende Util:
(1) Resize
Ändert die Größe des Icon
(2) PickColor
Setzt die aktive Farbe auf die Farbe des Pixel auf das als nächstes geklickt wird
(3) SetColor
Setzt die aktive Farbe über einen Farbauswahldialog
(4) Preview
Zeigt eine Vorschau des Bildes in Originalgröße.
Vorsicht: Sehr große Bilder passen nicht auf den Bildschirm.

Im Ordner "samples" sind Beispiele für Bilder von www.openclipart.org, die wir bearbeitet haben. Außerdem sind einige Eigenkonstruktionen dabei.

Es gibt change-log.txt. Dort sind die Neuerungen eingetragen.
